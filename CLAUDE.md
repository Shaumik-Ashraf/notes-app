## Architecture

**Rails 8.1** app with PostgreSQL, Hotwire (Turbo + Stimulus), and Devise authentication.

### Key Models

- **User** — Devise-managed; tracks sign-ins, supports lockout and password recovery
- **Note** — Belongs to User; `body` is encrypted at rest via Active Record Encryption; title is derived from first line of body

### Data Isolation

`NotesController` scopes all queries to `current_user` — `set_note` uses `find_by(id:, user: current_user)` so users can never access each other's notes.

### Real-time Updates

Create and update actions respond to Turbo Stream requests (`.turbo_stream.erb` templates) for partial page updates without full reloads. Multi-tab broadcasting is not yet implemented (noted in backlog).

### Multiple Databases

The app uses four PostgreSQL databases configured in `config/database.yml`:
- **primary** — application data (users, notes)
- **cache** — Solid Cache
- **queue** — Solid Queue background jobs
- **cable** — Solid Cable WebSocket state

### Auto-deletion

A rake task `notes:purge` deletes notes older than `PURGE_NOTES_AFTER_DAYS` days (default: 30). Intended to run via cron at 2am daily.

### Credential Setup

Development credentials (`config/credentials/development.yml.enc`) must contain:
- `secret_key_base`
- `database.username/password/host/port`
- `active_record_encryption` keys (generated by `bin/rails db:credentials:init`)
